PGVECTOR EXTENSION
================================================================================

What is pgvector?
├─ Open-source PostgreSQL extension
├─ Stores embeddings (vectors) in PostgreSQL
├─ Supports multiple distance operators
├─ HNSW index for fast approximate nearest neighbor search
└─ 150x speedup vs sequential scan

Installation (Supabase):
├─ CREATE EXTENSION IF NOT EXISTS vector;
├─ Create table with VECTOR(1536) column
├─ Create HNSW index
└─ Ready to use

Distance Operators:
├─ <-> : Euclidean distance (L2)
├─ <#> : Negative inner product (IP)
└─ <=> : Cosine distance

Vector Search Query:
SELECT id, content, embedding <-> query_embedding AS distance
FROM documents
ORDER BY embedding <-> query_embedding
LIMIT 10;

Performance:
├─ HNSW index: 150x speedup
├─ Supports 1M+ vectors
├─ Tunable recall vs speed
└─ <50ms query latency

================================================================================
FULL-TEXT SEARCH (FTS)
================================================================================

What is FTS?
├─ Native PostgreSQL full-text search
├─ Uses tsvector (text search vector) type
├─ GIN index for fast keyword search
└─ Supports multiple languages

Setup:
├─ Create tsvector column (auto-generated)
├─ Create GIN index
└─ Ready to use

Full-Text Search Query:
SELECT id, content, ts_rank_cd(fts, query) AS rank
FROM documents
WHERE fts @@ websearch_to_tsquery('english', 'search query')
ORDER BY rank DESC
LIMIT 10;

Features:
├─ Phrase search: "exact phrase"
├─ Boolean operators: & (AND), | (OR), \! (NOT)
├─ Prefix search: word:*
└─ Multiple languages supported

Performance:
├─ GIN index: <10ms for keyword search
├─ Efficient for large datasets
└─ Automatic index optimization

================================================================================
HYBRID SEARCH (pgvector + FTS)
================================================================================

Why Hybrid?
├─ Semantic search: Find by meaning (pgvector)
├─ Keyword search: Find by exact terms (FTS)
└─ Combined: Best of both worlds

Reciprocal Rank Fusion (RRF):
├─ Combines rankings from both methods
├─ Formula: 1 / (k + rank)
├─ k = smoothing constant (default 50)
└─ Weights: full_text_weight, semantic_weight

Hybrid Search Function:
├─ Executes FTS and semantic search separately
├─ Combines results using RRF
├─ Returns ranked results
└─ Tunable weights for each method

Performance:
├─ Combined: <100ms query latency
├─ Supports 1M+ documents
└─ Production-ready

IMPLEMENTATION STEPS
================================================================================

Step 1: Enable Extensions
CREATE EXTENSION IF NOT EXISTS vector;

Step 2: Create Tables
CREATE TABLE tools (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  name TEXT NOT NULL,
  description TEXT NOT NULL,
  fts TSVECTOR GENERATED ALWAYS AS (
    to_tsvector('english', name || ' ' || description)
  ) STORED,
  embedding VECTOR(1536),
  created_at TIMESTAMP DEFAULT NOW()
);

Step 3: Create Indexes
CREATE INDEX tools_fts_idx ON tools USING gin(fts);
