PROJECT OVERVIEW
================================================================================

PROJECT NAME: Semantic Tool Router with Auto-Discovery

OBJECTIVE:
Build an intelligent tool routing system that automatically selects appropriate
tools/CLIs based on user actions and prompts using Arch Router 1.5B, integrated
with semantic search (pgvector + FTS) and CLI hooks.

SCOPE:
✅ Local database stack (Docker Compose)
✅ Cloud database setup (Supabase)
✅ Hybrid search (pgvector + FTS)
✅ Auto-router (Arch Router 1.5B + MLX)
✅ CLI hooks integration (4 tools)
✅ Tool registry management

TIMELINE: 4 weeks
EFFORT: 410 hours
TEAM: 1-2 engineers
COST: $0 (open source + freemium)

================================================================================
TECHNOLOGY STACK
================================================================================

DATABASE LAYER:
├─ Local: PostgreSQL + Memgraph + Qdrant + Valkey (Docker)
├─ Cloud: Supabase + Qdrant Cloud + Upstash Redis
└─ Search: pgvector (semantic) + FTS (keyword) + Hybrid (RRF)

AUTO-ROUTER LAYER:
├─ Model: Arch Router 1.5B (Katanemo)
├─ Framework: MLX (Apple Silicon optimized)
├─ Language: Python 3.12+
└─ Performance: <50ms routing, 95%+ accuracy

CLI INTEGRATION LAYER:
├─ Cursor Agent (pre-discovery, post-selection, pre-execution, post-execution)
├─ Claude CLI (semantic-discovery, pre-commit, post-analysis)
├─ Auggie CLI (pre-discovery, tool-selection, post-action)
└─ Droid CLI (pre-test, test-discovery, post-test)

================================================================================
ARCHITECTURE
================================================================================

FLOW:
User Input (action + prompt)
    ↓
Semantic Pre-Discovery (Arch Router 1.5B)
    ↓
Tool Selection (from registry)
    ↓
Pre-Discovery Hook Execution
    ↓
CLI Command Execution (cursor-agent, claude, auggie, droid)
    ↓
Post-Execution Hook
    ↓
Result

COMPONENTS:
1. Router Engine (Arch Router 1.5B + MLX)
2. Tool Registry (YAML/JSON)
3. Semantic Search (pgvector + FTS)
4. Hook System (pre/post execution)
5. CLI Adapters (4 tools)
